# Проєкт: Класифікація Iris з MLflow

## Опис проєкту

Цей проєкт демонструє базовий приклад машинного навчання з використанням класичного датасету Iris. Модель логістичної регресії навчається класифікувати квіти ірисів на основі їх фізичних характеристик (довжина та ширина чашолистків і пелюсток).

Проєкт створено в рамках курсу з MLOps для демонстрації можливостей відстеження експериментів за допомогою MLflow.

## Структура проєкту

- `train.py` — скрипт для навчання моделі
- `models/` — директорія для збереження натренованих моделей
- `pyproject.toml` — конфігурація проєкту та залежності
- `uv.lock` — файл блокування залежностей

## Про датасет Iris

Датасет містить 150 зразків квітів ірисів трьох видів (Setosa, Versicolor, Virginica) з чотирма ознаками для кожного зразка. Це класичний датасет для навчання та тестування алгоритмів машинного навчання.

## Модель

Використовується модель **Logistic Regression** з бібліотеки scikit-learn з наступними гіперпараметрами:
- Solver: lbfgs
- Максимальна кількість ітерацій: 1000
- Random state: 8888

Дані розділяються на навчальну (80%) та тестову (20%) вибірки.

## Що можна відстежувати за допомогою MLflow

### Параметри моделі (Parameters)
У нашому проєкті це:
- **solver** — алгоритм оптимізації (lbfgs)
- **max_iter** — максимальна кількість ітерацій (1000)
- **random_state** — значення для відтворюваності (8888)

Також можна логувати параметри розбиття даних:
- **test_size** — розмір тестової вибірки (0.2)

### Метрики (Metrics)
Для нашої моделі класифікації можна відстежувати:
- **accuracy** — точність класифікації (наразі обчислюється в скрипті)

### Модель (Model)
- **LogisticRegression модель** — натренована модель scikit-learn
- **Сигнатура моделі** — опис входів (4 ознаки) та виходів (3 класи)
- **Model flavor** — sklearn модель для правильного завантаження

### Артефакти (Artifacts)
Для цього проєкту можна зберігати:
- **Серіалізована модель** — файл .joblib або інший формат

Вся ця інформація зберігається в MLflow і доступна через веб-інтерфейс, що дозволяє порівнювати різні запуски, знаходити найкращі гіперпараметри та відтворювати результати.

## Встановлення

Проєкт використовує `uv` для керування залежностями:

```bash
uv sync
```

